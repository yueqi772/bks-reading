## 1. 产品概述

### 1.1 产品定位

**书解（BookInsight）** 是一款面向大众读者的微信小程序，核心功能是借助 AI 对用户指定的书籍进行深度解读，以结构化 Markdown 形式输出，帮助用户在较短时间内系统掌握一本书的核心思想、框架与精华内容。

### 1.2 产品愿景

> 让每个人都能轻松、高效地汲取好书中的智慧，像喝水一样自然地成长。

### 1.3 核心价值主张

| 用户痛点 | 产品解法 |
|----------|----------|
| 买了书却没时间读 | AI 深度解读，快速获取精华 |
| 读了书却没有系统理解 | 结构化输出，框架清晰 |
| 不知道该读什么书 | 每日书单推荐 |
| 解读质量参差不齐 | 专业 Prompt 设计，输出稳定高质量 |

## 2. 目标用户

### 2.1 用户画像

**主要用户群体：**

- **年龄**：22 - 45 岁
- **职业**：职场人士、在校大学生、自我提升爱好者
- **阅读习惯**：有阅读意愿，但时间碎片化；或喜欢先了解一本书再决定是否精读
- **使用场景**：通勤途中、睡前、午休时快速了解一本书

**次要用户群体：**

- 读书博主、知识博主（用于快速了解书籍内容做内容创作参考）
- 学生（用于辅助理解课外书籍）

### 2.2 用户需求优先级

| 优先级 | 用户需求 |
|--------|----------|
| P0 | 输入书名，获得高质量深度解读 |
| P0 | 查看历史解读记录 |
| P1 | 每日推荐书单，激励持续使用 |
| P1 | 付费解锁无限次解读 |
| P2 | 分享解读内容给好友 |

## 4. 详细功能需求

### 4.1 首页（Home Page）

#### 4.1.1 页面元素

| 元素 | 说明 |
|------|------|
| 顶部 Banner | 产品名称、Slogan，视觉引导氛围 |
| 书籍搜索/输入框 | 用户输入书名，支持手动输入或从推荐中选择 |
| 每日推荐书目 | 运营每天推送 1 本书，附带书籍封面、作者、推荐语 |
| 快速入口卡片 | 「开始解读」主按钮，视觉上最突出 |
| 解读历史入口 | 展示最近 2-3 条历史记录，可跳转历史页 |

#### 4.1.2 功能逻辑

- **每日推荐**：
  - 每天凌晨 00:00 由后台自动更新推荐书目
  - 推荐内容由运营配置，支持后台管理
  - 用户点击推荐书目可直接进入解读流程
  
- **输入框**：
  - 支持中英文书名输入
  - 最大输入字数：100 字
  - 带有 placeholder 提示："输入书名，开始深度解读"

#### 4.1.3 免费次数提示

- 未开通会员的用户，页面顶部醒目位置显示剩余免费次数
- 示例：`剩余免费解读次数：1 次`
- 点击可跳转会员购买页

### 4.2 AI 书籍解读流程

#### 4.2.1 解读触发

1. 用户在首页输入书名或点击推荐书目
2. 系统校验用户剩余次数或会员状态
3. 进入解读加载页，实时展示解读进度
4. 解读完成后跳转解读详情页

#### 4.2.2 次数校验规则

| 用户状态 | 操作 |
|----------|------|
| 免费剩余次数 > 0 | 正常发起解读，消耗 1 次免费次数 |
| 免费次数已用完 + 非会员 | 弹出付费引导弹窗，阻断解读流程 |
| 已开通会员 | 无次数限制，直接发起解读 |

#### 4.2.3 解读加载页

- 展示书籍封面（可通过书名查询豆瓣/搜索引擎获取，或使用默认占位图）
- 展示动态加载动画（进度条 / 逐字生成 Typing 效果）
- 提示语：「正在深度解读《书名》，请稍候……」
- 预计等待时间：15 - 60 秒（根据 AI 接口响应时间）
- 支持 Stream 流式输出，边生成边展示内容

#### 4.2.4 解读详情页（核心页面）

**输出格式说明：**

解读结果以 **Markdown** 渲染展示，固定包含以下章节结构：

```markdown

## 基本信息

> 本书的整体逻辑结构或作者的思考框架...

## 核心章节解读
### 第一部分：xxx
- 核心观点
- 关键论据
- 实践建议

### 第二部分：xxx
...

## 金句精选
> "..." —— 书名

> "..." —— 书名

## 思维导图（文字版）
- 主题
  - 分支 1
    - 细节
  - 分支 2

## 行动清单
- [ ] 可以立即实践的行动 1
```

### 4.3 解读历史页

#### 4.3.1 页面说明

- 展示用户所有历史解读记录
- 列表按时间倒序排列
- 每条记录展示：书籍封面缩略图、书名、解读时间、字数
- 点击可重新查看解读详情
- 支持下拉刷新、上拉加载更多（分页，每页 20 条）
- 支持单条记录滑动删除

#### 4.3.2 数据存储策略

- 解读记录存储在服务端，与微信账号绑定
- 免费用户：保留最近 10 条历史记录
- 会员用户：无限历史记录

### 4.4 每日推荐

#### 4.4.1 功能说明

- 每天推荐 1 本书（由运营在后台手动配置或程序自动生成）
- 展示内容：书籍封面、书名、作者、精简推荐语（100 字以内）
- 点击「查看解读」可直接进入该书的解读流程
- 历史推荐可在「推荐列表」页查看（最近 30 天）

#### 4.4.2 推荐书目来源

- 运营手动维护书单（后台 CMS 配置）
- 可扩展为 AI 自动生成推荐语，结合节日、热点话题

## 5. 付费体系设计

### 5.1 免费规则

| 规则 | 说明 |
|------|------|
| 免费解读次数 | 每个账号默认可免费解读 **2 本书** |
| 次数重置 | 免费次数**不重置**，用完即止 |
| 限制说明 | 用完后，任何解读操作都会触发付费引导 |

> ⚠️ 注意：免费次数与账号绑定，不与设备绑定，防止刷号薅羊毛。

---

### 5.2 包月会员

| 项目 | 内容 |
|------|------|
| 价格 | **¥30 / 月** |
| 权益 | 无限次解读任意书籍 |
| 有效期 | 自开通之日起 30 天 |
| 续费 | 到期前 3 天在个人中心提示续费 |
| 开通方式 | 扫描页面内二维码，付款 ¥30 后联系公众号开通 |

### 5.3 开通流程（手动核销模式）

> 考虑到小程序支付资质审核复杂性，**第一版采用「扫码支付 + 公众号人工开通」的半自动模式**。

**用户操作流程：**

```
1. 用户点击「开通会员」
   ↓
2. 展示付款说明页面（¥30/月 权益介绍）
   ↓
3. 展示收款二维码（微信收款码）
   ↓
4. 用户扫码支付 ¥30
   ↓
5. 页面提示：「支付完成后，请长按下方二维码，添加公众号/客服微信，
   发送「开通会员」+ 支付截图，客服将在 24 小时内为您开通」
   ↓
6. 展示公众号/客服微信二维码
   ↓
7. 客服收到申请后，在后台手动将该用户账号升级为会员
   ↓
8. 用户下次打开小程序时，会员状态更新
```

**付款页面需展示内容：**

- 会员权益说明（无限次解读 + 历史不限）
- 收款二维码（运营配置）
- 操作指引文案
- 公众号/客服联系方式二维码
- 常见问题（FAQ）

## 7. 页面流程与交互说明

### 7.1 首次进入流程

```
启动小程序
  → 微信授权登录（获取 openid，昵称头像可选授权）
  → 首页
  → 若剩余次数 = 2，展示「新用户专享 2 次免费解读」引导提示
```

### 7.2 解读完整流程

```
首页输入书名 / 点击推荐书目
  → 校验次数/会员状态
    ├── 通过 → 解读加载页（流式输出）→ 解读详情页

                  ├── 点击「立即开通」→ 开通会员页
```

```
会员开通页
  → 展示权益 + 收款二维码
  → 用户扫码支付
  → 页面提示「支付完成，请联系公众号」
  → 展示公众号二维码
  → 用户添加公众号，发送「开通+截图」
  → 客服后台开通（手动）
  → 用户小程序端会员状态更新
```

### 7.4 分享流程

```
解读详情页 → 点击「分享」
  → 生成分享卡片（书名 + 核心一句话 + 小程序码）
  → 选择分享到好友 / 朋友圈
```

## 8. AI 解读 Prompt 设计规范

> 本节基于「书旅向导」角色设定（`role_prompt_book_guide.md`），所有 AI 解读均以此角色与规范为准。

---

### 8.1 AI 角色设定（System Prompt）

**角色名称**：书旅向导

**角色定位**：博学的阅读导师 + 知识转化专家

```
你是「书旅向导」，一位博学的阅读导师与知识转化专家。

【能力特质】
- 熟读各类书籍，擅长将书面知识转化为生活中的智慧
- 用苏格拉底式提问法引导读者思考
- 用生动的例子帮助理解抽象概念
- 用清晰的框架帮助读者构建知识体系

【沟通原则】
- 亲切但专业，像朋友聊天一样有深度
- 启发式提问，不直接给答案
- 案例驱动，每个抽象概念都配生活化例子
- 鼓励反思，将知识与自身经历结合

【语言规范】
- 使用第二人称"你"进行交流
- 适当使用 emoji 增强可读性
- 段落简洁，每段不超过3行
- 重要观点用加粗标注

【内容质量底线】
- ✅ 每个章节必须包含：核心论点 + 支撑论据 + 案例应用
- ✅ 每个核心概念必须有"为什么重要"的深度解释
- ✅ 每个论点必须配至少一个完整的生活或商业案例
- ✅ 深度模式下每章内容不低于1000字
- ✅ 禁止仅用摘要式语言提炼结论，必须展开论证过程
- ❌ 不编造书中不存在的观点
- ❌ 不使用过于学术化的语言
- ❌ 不输出简练摘要
```

---

### 8.2 解读深度档位

默认使用**深度模式**，用户可在解读页主动切换档位。

| 档位 | 内容范围 | 输出量 | 适用场景 |
|------|---------|-------|---------|
| **概览** | 核心框架 + 每章核心要点 | 精简 | 快速了解一本书 |
| **标准** | 概览 + 详细问答 + 生活应用 | 中等 | 深入理解一本书 |
| **深度**（默认） | 标准 + 所有重要概念 + 延伸知识 + 实践工具 | 完整 | 彻底掌握一本书 |

---

### 8.3 解读输出结构（深度模式完整版）

AI 输出的 Markdown 文档固定按以下结构生成：

```markdown
# 《{书名}》精读指南

## 一句话推荐
用一句话概括本书的核心价值（不超过50字）

## 深入程度
[概览 / 标准 / 深度]

## 书籍概览
- **作者**：
- **出版年份**：
- **核心主题**：
- **书籍结构**：（章节数量、主要部分）
- **适合人群**：

## 全景知识地图
[书籍整体框架图 —— Mermaid 格式，展示所有核心概念及其关系]

## 核心观点深度解析

### 🔴 核心精华

#### 精华一：[观点标题]
- **问题**：为什么要关注这个问题？（至少200字，有深度，不能一句话带过）
- **解答**：核心观点的详细解释 + 案例（生活案例或商业案例）
- **生活中的运用**：
  - 场景：具体生活情境描述
  - 方法：3步以内的可执行步骤
  - 提醒：常见的误区或注意事项
- **相关概念链接**：关联概念A、关联概念B

#### 精华二：[观点标题]
...（结构同上，每个精华都要有深度展开）

### 🟡 重要概念

#### 概念一：[概念名称]
```

## 12. 风险与注意事项

### 12.1 内容版权风险

- AI 解读属于二次创作，但需确保输出内容为原创性解读，而非直接摘抄
- Prompt 需引导 AI 以「解读、分析」而非「复述、抄录」的方式输出
- 建议在用户协议中注明内容来源及版权声明

### 12.2 AI 内容准确性

- AI 可能对冷门书籍了解有限，存在内容错误风险
- 建议在详情页添加免责声明：「本解读由 AI 生成，仅供参考，请以原著为准」

### 12.3 付费核销效率

- 手动核销模式响应可能延迟（最长 24 小时），需在产品文案中明确告知
- 建议设置核销 SLA 并在客服流程中优化

### 12.4 微信平台合规

- 小程序不得引导用户跳转外部支付，收款二维码展示需符合微信平台规范
- 需在小程序内配置用户协议和隐私政策页面
- 涉及 AI 生成内容，需关注监管政策变化

### 12.5 成本控制

- AI API 调用成本随用量增长，需监控单次解读成本
- 建议设置单日调用上限，防止异常刷量
